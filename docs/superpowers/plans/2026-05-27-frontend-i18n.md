# Frontend i18n (EN/FR) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a lightweight custom i18n layer to the React frontend so the whole UI is available in English (default) and French, toggled by an always-visible switcher and persisted in `localStorage`.

**Architecture:** A React Context (`LanguageProvider`) holds the current `lang`, initialised from `localStorage` (fallback `"en"`). Two typed dictionaries (`en.ts` source of truth, `fr.ts` keyed by `keyof typeof en`) back a `useT()` hook exposing `t(key, params?)`, `lang`, `setLang`. Components render `t("key")` instead of hardcoded French. A `LanguageSwitcher` mounted once in `App.tsx` flips the language everywhere.

**Tech Stack:** React 19, TypeScript, Vite, Vitest + Testing Library (globals, jsdom). Spec: `docs/superpowers/specs/2026-05-27-frontend-i18n-design.md`.

**Commit rule:** No `Co-Authored-By: Claude` trailer in any commit.

- [ ] **Step 4: Create the context + hook**

`frontend/src/i18n/LanguageContext.tsx`:

```tsx
import {
  createContext,
  useContext,
  useEffect,
  useState,
  type ReactNode,
} from "react";
import { en } from "./en";
import { fr } from "./fr";
import type { Lang, TranslationKey } from "./types";

const dicts = { en, fr } as const;
const STORAGE_KEY = "lang";

function readInitialLang(): Lang {
  const stored = localStorage.getItem(STORAGE_KEY);
  return stored === "fr" || stored === "en" ? stored : "en";
}

export interface LanguageContextValue {
  lang: Lang;
  setLang: (l: Lang) => void;
  t: (key: TranslationKey, params?: Record<string, string | number>) => string;
}

const LanguageContext = createContext<LanguageContextValue | null>(null);

export function LanguageProvider({ children }: { children: ReactNode }) {
  const [lang, setLang] = useState<Lang>(readInitialLang);

  useEffect(() => {
    localStorage.setItem(STORAGE_KEY, lang);
  }, [lang]);

  const t = (
    key: TranslationKey,
    params?: Record<string, string | number>
  ): string => {
    const template = dicts[lang][key] ?? dicts.en[key] ?? key;
    if (!params) return template;
    return template.replace(/\{(\w+)\}/g, (_, k) =>
      k in params ? String(params[k]) : `{${k}}`
    );
  };

  return (
    <LanguageContext.Provider value={{ lang, setLang, t }}>
      {children}
    </LanguageContext.Provider>
  );
}

export function useT(): LanguageContextValue {
  const ctx = useContext(LanguageContext);
  if (!ctx) throw new Error("useT must be used within a LanguageProvider");
  return ctx;
}
```

- [ ] **Step 5: Create the plural helper**

`frontend/src/i18n/plural.ts`:

```ts
import type { TranslationKey } from "./types";

type TFn = (key: TranslationKey, params?: Record<string, string | number>) => string;

export function plural(t: TFn, n: number, base: "mistake" | "entity"): string {
  const key = (n === 1 ? `${base}_one` : `${base}_other`) as TranslationKey;
  return t(key, { n });
}
```

**Recipe (apply to each component listed in the task):**
